As a library to manage state, StateMachine does this via an event-based system which allows any state machine in your system to remain decoupled from your application code.

It's not much effort to

  1. HTML elements to represent state, and some buttons to call actions
  2. Global CSS styles to make the page look pretty, with some local styles if required
  3. A page script to configure the states and handlers and create the StateMachine
  4. A call to a simple reusable setup function which listens to StateMachine events and updates the UI on changes

The code for the example above is:

var fsm = new StateMachine({
    transitions: [
        'next : a > b > c',
        'back : a < b < c'
    ]
});

setup(fsm);

Check out demo/assets/setup.js to view the very simple UI code.

All the code for each example is in its file, and check the console for debug output.
