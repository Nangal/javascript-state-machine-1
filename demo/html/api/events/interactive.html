This page allows you to test out some event handler ids of your own.

A StateMachine instance is instantiated with the following properties available:

  • States: a, b, c, d, e, exit
  • Actions: back, next, quit

The following grammar is available:

  • namespace.foo namespaced events, for system and transition
  • foo tokens, including namespace aliases, actions and states
  • :foo event types, for global, action or state events, such as "start" and "enter"
  • @foo action, filter to actions only
  • #foo state, provided in case you need to disambiguate tokens
  • (foo bar baz) multiple grammars, which are expanded to multiple paths

Type above or use the examples to test out some matches!

The "Invalid states / actions" option a value you can set in config to handle missing states or actions as you add event handlers.
